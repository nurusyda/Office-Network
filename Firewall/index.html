Firewall Configuration Checklist

ASA Firewall Essential Checks

1. Interface Verification

show interface
show nameif
  • Confirm security levels (inside=100, outside=0)
  • Verify IP addresses match network topology

2. Access Control Lists (ACLs)

show access-list
show access-group
  • Replace permissive rules with least privilege: 
    access-list INSIDE_OUT extended permit tcp 192.168.1.0 255.255.255.0 any eq 80
access-list INSIDE_OUT extended permit tcp 192.168.1.0 255.255.255.0 any eq 443
  • Ensure ACLs are applied to correct interfaces

3. NAT Configuration

show nat
show xlate
  • Verify NAT overload is configured: 
    nat (inside,outside) source dynamic 192.168.1.0/24 interface
  • Check for proper address translation

4. Routing Verification

show route
ping outside 192.168.0.1
  • Confirm default route exists: 0.0.0.0/0 to 192.168.0.1
  • Test next-hop reachability

5. Security Policies

show running-config ssh
show running-config http
  • Restrict management access: 
    ssh 192.168.1.0 255.255.255.0 inside
http 192.168.1.0 255.255.255.0 inside
  • Enable protocol inspection: 
    policy-map global_policy
 class inspection_default
  inspect http
  inspect dns

Troubleshooting Tips

Firewall-Specific Checks

  • Monitor live connections: 
    show conn
  • Check ACL hit counts: 
    show access-list INSIDE_OUT
  • Verify NAT translations: 
    show xlate detail
  • Test with packet tracer: 
    packet-tracer input inside tcp 192.168.1.2 12345 8.8.8.8 53

Troubleshooting Tips

  • Verify PC connectivity to its gateway: ping 192.168.1.1
  • Check NAT rules on Router2 with show ip nat translations
  • Confirm ASA routes with show route
  • Test DNS resolution from routers: nslookup cisco.com 8.8.8.8
